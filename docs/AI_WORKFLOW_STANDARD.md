# AI Workflow Standard

Этот документ фиксирует обязательный стиль работы для всех дальнейших изменений в проекте SharipovAI.

## Главный принцип

Нельзя просто внедрить код и сразу сказать, что всё готово.

Каждое изменение должно проходить цикл:

```text
внедрить
самостоятельно проверить
найти слабые места
исправить найденное
добавить или обновить тесты
только потом отчитаться
```

## Что должен делать AI/разработчик после изменения

После каждого блока работы нужно проверить:

```text
1. Не сломаны ли импорты.
2. Не конфликтуют ли новые файлы со старыми.
3. Есть ли тест на новую функцию.
4. Не раскрываются ли пароли, токены, password_hash или secrets.
5. Не получил ли обычный user доступ к admin/security-разделам.
6. Не добавлены ли английские UI-слова там, где интерфейс должен быть русским.
7. Есть ли документация, если изменился запуск или поведение системы.
8. Не надо ли сразу исправить найденное слабое место.
```

## Обязательное поведение

Если во время самопроверки найдена ошибка, слабое место или неаккуратный тест, нужно исправить это сразу, не ждать отдельной команды.

Пример правильного отчёта:

```text
Сделал блок.
Проверил связность файлов.
Нашёл проблему в тесте.
Исправил.
Добавил документацию.
Вот коммиты.
```

Пример неправильного отчёта:

```text
Сделал. Проверь сам.
```

## Тесты

Для Python/FastAPI части проекта стандартная проверка:

```powershell
python -m pytest
```

Если добавлена новая логика, к ней должен быть тест:

```text
новый сервис -> unit test
новый API -> integration test
новая security-логика -> negative test тоже обязателен
новый UI-фильтр -> тест на HTML/видимость
```

## Security-first правила

Нельзя коммитить:

```text
.env
пароли
токены
секреты
реальные ключи API
password_hash из runtime-файлов
runtime JSON с пользователями
```

Нельзя ослаблять защиту ради удобства.

Если есть сомнение, выбирать более безопасный вариант:

```text
отдельный слой
отдельный entrypoint
документация
тест
```

## Текущий правильный production/dev entrypoint

Для полной защищённой версии сейчас использовать:

```powershell
python -m uvicorn dashboard.user_admin_app:app --reload
```

Он включает:

```text
login
access requests
approval flow
temporary password change
login lockout
roles admin/user
admin-only security center
role-aware menu
user management
```

## Стандарт ответа пользователю

В отчёте нужно писать:

```text
что внедрено
какие файлы изменены или созданы
какие тесты добавлены
что было найдено самопроверкой
что исправлено сразу
какая команда запуска теперь правильная
```

Если автоматическая проверка через GitHub Actions или локальный pytest ещё не подтверждена, нельзя писать, что всё точно прошло. Нужно писать честно:

```text
Добавлены тесты. Их должен выполнить CI или локальная команда python -m pytest.
```
