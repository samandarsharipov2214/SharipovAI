# SharipovAI Web — каноническая спецификация

Этот документ является единственным источником требований для всех чатов, Codex и разработчиков проекта. Пользователь не должен повторять уже согласованные пожелания.

## 1. Бренд
- Правильное название везде: **SharipovAI**.
- Использовать утверждённый логотип SA с биржевыми свечами и слоганом `SMARTER. DATA. DECISIONS.`.
- Не заменять логотип текстовой заглушкой или другим вариантом.

## 2. Архитектура и стоимость
- Новый интерфейс создаётся с нуля; старый фронтенд не ремонтируется и не развивается.
- Backend, API, Bybit, Telegram, AI-модули и данные сохраняются.
- Сайт и backend работают на одном существующем Render Web Service, без второго платного сервиса.
- Публичный адрес остаётся текущим адресом проекта.
- UI не должен исчезать при ошибках backend или биржи.

## 3. Визуальный стиль
- Тёмный премиальный интерфейс: глубокий чёрно-синий фон, металл, стекло, неоновый синий.
- Зелёный — прибыль и исправные системы; красный — убыток и авария; жёлтый — предупреждение.
- Стиль профессионального AI Trading OS, а не технической админки.
- Никакого сырого JSON, traceback и пустых чёрных экранов.
- Понятные карточки, таблицы, графики, статусы и объяснения простым языком.

## 4. Обязательные разделы
1. Mission Control / Обзор
2. Рынок и живые котировки
3. AI-решение и консенсус
4. Портфель
5. Сделки и журнал исполнения
6. Bybit — баланс, позиции, ордера, история, комиссии, статус API
7. AI-боты — карточка каждого действующего AI без дубликатов
8. AI Chat
9. Новости и оценка влияния
10. Risk Center и Kill Switch
11. Learning OS
12. Генеральный контроль
13. Evidence Vault
14. Virtual Account / Paper Trading
15. Отчёты
16. Настройки

## 5. Реальные данные
- Котировки, движение рынка, баланс, позиции и ордера берутся только из реальных API/WebSocket или явно маркируются как demo/paper.
- Запрещено показывать выдуманные значения как реальные.
- Показывать время последнего обновления и источник данных.
- При недоступности API отображать понятный статус и последнюю подтверждённую информацию.

## 6. Главная панель
- Общий капитал, доступные средства, PnL за день/неделю, открытые позиции, риск.
- График портфеля и выбранного рынка.
- Текущее AI-решение, уверенность, причины и голоса модулей.
- Последние сделки и текущая деятельность AI.
- Статусы backend, PC Agent, backup, Bybit, Telegram, market stream и AI-сети.

## 7. UX
- Все вкладки видимы и работают; меню не зависит от успешной загрузки JavaScript/API.
- Минимум кликов, без лишних модальных окон и сложных форм.
- Адаптивность для Windows, iPhone и других мобильных устройств.
- Быстрая загрузка; ошибки одного блока не ломают всю страницу.
- Пользовательские сообщения — понятные, без внутренних исключений.

## 8. Безопасность
- Авторизация fail-closed.
- Секреты и API-ключи никогда не выводятся в UI, логах или ответах.
- Вывод средств через API не используется.
- Реальная торговля, Testnet и Mainnet включаются только отдельными проверенными этапами.
- Kill Switch и риск-лимиты отображаются явно.

## 9. Правила разработки
- Перед изменениями учитывать контекст всего проекта и этот документ.
- Не задавать пользователю вопросы, ответы на которые уже зафиксированы.
- Не создавать дублирующие AI или параллельные реализации одной функции.
- Использовать отдельные ветки/PR для крупных изменений; Codex применять для разработки и review, когда доступен.
- Не переключать production до успешных проверок.
- После каждого изменения проверять lint, build, backend tests, runtime smoke, API и визуальный результат.
- Не говорить «готово», пока публичный сайт фактически не открыт и не проверен.

## 10. Критерий готовности
- Все обязательные разделы реализованы и доступны.
- Название и логотип правильные во всех местах.
- Нет пустого экрана, пропавшего меню, сырого JSON или выдуманных live-данных.
- Сайт работает на одном существующем Render-сервисе.
- Проверены desktop/mobile, API fallback, авторизация, runtime и публичный URL.
