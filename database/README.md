# Единая база SharipovAI

`database/migrations/0001_unified_core.sql` — каноническая PostgreSQL-схема для всех чатов и девяти AI-органов.

## Обязательные правила

- Production работает только с PostgreSQL через `DATABASE_URL`.
- Отсутствующий или небезопасный URL блокирует database-функции.
- Секреты подключения нельзя возвращать через API или писать в логи.
- TradingCandidate, execution journal и private order state хранят устойчивые идентификаторы.
- Неопределённое состояние исполнения всегда означает `BLOCK`.
- JSON-файлы не удаляются, пока данные не перенесены и не проверены.

## Порядок внедрения без остановки проекта

1. Создать PostgreSQL и задать `DATABASE_URL` только в секретах Render/локального узла.
2. Выполнить `python -m database.migrate`.
3. Проверить `UnifiedStore.health()`.
4. Включить dual-write: PostgreSQL + существующий JSON.
5. Сравнить записи и контрольные суммы.
6. Перевести чтение на PostgreSQL по одному домену:
   - общая память чатов;
   - состояния AI-органов;
   - новости и рынок;
   - портфель;
   - кандидаты и execution evidence.
7. Оставить JSON как резервный экспорт, а не источник истины.

## Переменные окружения

- `DATABASE_URL` — обязательный PostgreSQL URL.
- В CI без реальной базы выполняются contract-тесты схемы и fail-closed конфигурации.
- Интеграционные DB-тесты должны использовать отдельную временную базу.

## Запреты

- Нельзя использовать одну таблицу без `environment` для testnet и mainnet execution state.
- Нельзя автоматически повторять заявку после timeout без reconciliation.
- Нельзя хранить API-ключи и пароли в JSONB payload или project memory.
